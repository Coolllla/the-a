# 踩坑点

 2. right: 10% 不生效的原因

  这是 next/image 的 fill 模式特有的坑。

  fill 会给底层 <img> 打上inline style：

  <img style="position: absolute; inset: 0; width: 100%; height: 100%;">

  inline style 优先级高于你的 module scss，所以你写在 .pic 上的 right: 10% 根本盖不掉它的 right: 0。再加上
  width: 100%，图片被强制铺满父容器，right 就算盖住了也没视觉效果。

  正确做法：定位逻辑放在外层 wrapper 上，<Image fill> 只负责"填满 wrapper"。改造 CharacterImg：

  function CharacterImg({ imgSrc, name, zIndex, ref }: Props) {
    return (
      <div
        ref={ref}                                  // ← ref 改挂 wrapper
        className={styles.slot}
        data-character={name}
        style={{ "--z": zIndex } as React.CSSProperties}
      >
        <Image
          src={imgSrc}
          alt={name}
          fill
          style={{ objectFit: "contain" }}
          className={styles.pic}
        />
      </div>
    );
  }

  .slot {
    position: absolute;             // 让 right/left/top 都能用
    inset: 0;                       // 默认铺满，单个角色再用 data-character 覆盖
    z-index: var(--z, 1);

    &[data-character="bg"] {
      z-index: -1;
    }
    
    &[data-character="dorath"] {
      left: auto;                   // ← 关键：清掉 inset 里的 left
      right: 10%;
      width: 50%;                   // 给个宽度，不然 right 没参考
    }
  }

  .pic {
    object-position: center bottom;
  }

  ▎ 顺便：ref 类型也要从 HTMLImageElement 改成 HTMLDivElement，对应 wrapper 的 div。useParallax 里传的 selector
  ▎ 类型也跟着改 —— GSAP 动 div 还是 img 没差，反正是动 transform。

  ▎ ⚠️ inset: 0 + right: 10% 不直接生效是因为 inset: 0 等价于同时设了 left: 0; right:
  ▎ 0，特异性相同时后写的赢，但这俩都在同一条 selector 内 —— 所以要么用 inset: auto 重置，要么显式写 left:
  ▎ auto。
