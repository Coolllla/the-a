# rAF 是什么

rAF 是 `requestAnimationFrame` 的简称，是浏览器提供的一个 JS API，用于告诉浏览器"在下一次重绘之前执行这个回调函数"。

**核心特点：**
- 由浏览器的刷新率驱动（通常 60fps，即约每 16.7ms 触发一次；高刷屏会更快）
- 与浏览器的渲染流程同步，避免了 `setTimeout`/`setInterval` 可能出现的掉帧、卡顿、不同步问题
- 当页面不可见（比如切到后台标签）时会自动暂停，节省性能和电量

**典型用法：**

```js
function animate() {
  // 更新动画状态，比如移动元素位置
  el.style.transform = `translateX(${x}px)`;
  x += 1;

  requestAnimationFrame(animate); // 请求下一帧继续执行
}

requestAnimationFrame(animate);
```

**常见场景：**
- 手写 CSS 动画/过渡之外的复杂动画（游戏、Canvas 绘制、滚动特效）
- 高频事件（如 `scroll`、`resize`）的节流，避免一帧内重复计算
- 需要读取布局信息再修改样式时，减少强制同步布局（layout thrashing）

**对比：**
- `setTimeout(fn, 16)`：时间不精确，不与屏幕刷新同步，可能丢帧
- `rAF`：与显示器刷新同步，更平滑，且页面隐藏时自动暂停

配套还有 `cancelAnimationFrame(id)` 用来取消已请求的回调。

# 逐像素 hover 检测（alpha 位图 hit-test）方案

1. 给每张图（除背景）建 alpha 位图 —— 图片 onLoad 后一次性画到 offscreen canvas，提取 alpha 通道存 Uint8Array，之后就不再用 canvas。
2. 在 HomeV1 里挂 onMouseMove 到 homeview 容器，用 requestAnimationFrame 节流。
3. 每次事件按 zIndex 从高到低遍历角色——第一个 alpha 命中的就是当前 hover 目标。
4. 换算鼠标坐标：容器坐标 → slot 内坐标 → 图片实际绘制矩形内坐标 → 图片原始像素坐标。
5. setState(hoveredCharacter) 传给 NameCard 决定显示什么名字。

---

## 四个容易踩的坑

| 坑 | 说明 |
|---|---|
| object-fit: contain 的换算 | slot 是全屏，但图片只居中/贴底渲染。要根据 naturalWidth/Height vs slot 宽高比算出实际绘制矩形（并且区分 object-position: bottom vs center） |
| transform: scale(1.05) | 图片被放大 5%，getBoundingClientRect() 拿不到"放大之后"的图片矩形，需要在换算时把 draw rect 用 scale 展开 |
| useParallax 也在动 transform | slot 位置每帧在变——每次 mousemove 都要 getBoundingClientRect() 实时读，不能缓存 |
| 异步加载 | next/image 是异步的，img.decode() 完成前 canvas 画出来是空的。挂 onLoad 且判断 complete && naturalWidth > 0 |

## 核心 demo（可直接抄，其他你自己拼）

### 1. alpha 位图建表（一次性）

```ts
// useAlphaMap.ts
import { useEffect, useState } from "react";

export type AlphaMap = { width: number; height: number; data: Uint8Array };

export function buildAlphaMap(img: HTMLImageElement): AlphaMap {
  const canvas = document.createElement("canvas");
  canvas.width = img.naturalWidth;
  canvas.height = img.naturalHeight;
  const ctx = canvas.getContext("2d", { willReadFrequently: false })!;
  ctx.drawImage(img, 0, 0);
  const raw = ctx.getImageData(0, 0, canvas.width, canvas.height).data;
  const alpha = new Uint8Array(canvas.width * canvas.height);
  for (let i = 0; i < alpha.length; i++) alpha[i] = raw[i * 4 + 3];
  return { width: canvas.width, height: canvas.height, data: alpha };
}
```

### 2. hit-test 核心逻辑

```ts
// hitTest.ts
type ObjectPos = "center" | "bottom";

function computeContainRect(
  slotW: number, slotH: number,
  imgW: number, imgH: number,
  pos: ObjectPos
) {
  const slotRatio = slotW / slotH;
  const imgRatio = imgW / imgH;
  const drawW = imgRatio > slotRatio ? slotW : slotH * imgRatio;
  const drawH = imgRatio > slotRatio ? slotW / imgRatio : slotH;
  const x = (slotW - drawW) / 2;
  const y = pos === "bottom" ? slotH - drawH : (slotH - drawH) / 2;
  return { x, y, w: drawW, h: drawH };
}

export function hitTestAlpha(
  clientX: number, clientY: number,
  slotEl: HTMLElement,
  alphaMap: AlphaMap,
  opts: { objectPosition?: ObjectPos; scale?: number; threshold?: number } = {}
): boolean {
  const { objectPosition = "center", scale = 1, threshold = 20 } = opts;
  const rect = slotEl.getBoundingClientRect();
  const lx = clientX - rect.left;
  const ly = clientY - rect.top;

  // contain 绘制矩形
  const d = computeContainRect(rect.width, rect.height, alphaMap.width, alphaMap.height, objectPosition);

  // 再考虑 transform: scale(N)——以中心为原点放大
  const cx = d.x + d.w / 2, cy = d.y + d.h / 2;
  const sw = d.w * scale, sh = d.h * scale;
  const sx = cx - sw / 2, sy = cy - sh / 2;
  if (lx < sx || lx > sx + sw || ly < sy || ly > sy + sh) return false;

  // 换算到图片原始像素
  const px = Math.floor(((lx - sx) / sw) * alphaMap.width);
  const py = Math.floor(((ly - sy) / sh) * alphaMap.height);
  return alphaMap.data[py * alphaMap.width + px] > threshold;
}
```

### 3. HomeV1 里的连接（骨架，你补细节）

```ts
const [hovered, setHovered] = useState<string | null>(null);
const alphaMapsRef = useRef<Record<string, AlphaMap>>({});
const rafIdRef = useRef<number | null>(null);

// 图片加载后建 alpha 位图（在 CharacterImg 里 onLoad 触发，回调到父级）
const handleImgLoad = (name: string, img: HTMLImageElement) => {
  alphaMapsRef.current[name] = buildAlphaMap(img);
};

const onMove = (e: React.MouseEvent) => {
  const cx = e.clientX, cy = e.clientY;
  if (rafIdRef.current) return;
  rafIdRef.current = requestAnimationFrame(() => {
    rafIdRef.current = null;
    // 按 zIndex 从高到低
    const sorted = [...CHARACTERS].filter(c => c.name !== "bg")
      .sort((a, b) => b.zindex - a.zindex);
    for (const c of sorted) {
      const slot = charactersRef.current[c.name];
      const map = alphaMapsRef.current[c.name];
      if (!slot || !map) continue;
      if (hitTestAlpha(cx, cy, slot, map, {
        objectPosition: "bottom",  // 你 SCSS 里是 bottom
        scale: 1.05,               // 你的 transform: scale
        threshold: 20,
      })) {
        setHovered(c.name);
        return;
      }
    }
    setHovered(null);
  });
};

<div className={styles.homeview} onMouseMove={onMove} onMouseLeave={() => setHovered(null)}>
```

## 一些注意

- willReadFrequently 我设了 false，因为只读一次像素而不是频繁建表，就要设 true；
- 内存：一张 1000×1500 的图 alpha 位图 = 1.5MB。四个角色 6MB 左右，可接受；
- transform: scale 我这里假设 origin 是 center（默认值）。你如果之后改成 bottom 之类需要相应调整；
- 视差里如果不只是 translate 还有 rotate() 依然对——它拿的是变换后的最终矩形。

# 上面这段方案的通俗解释

好，我用更直白的方式给你捋一遍。这套东西解决的核心问题是：**鼠标放在角色图片的"透明区域"（比如衣服边缘之外的空白）时，不应该触发 hover；只有放在角色"实际画的像素"上才算 hover。**

## 为什么需要这么复杂

如果你角色图是矩形 PNG（周围是透明背景），最简单的做法是给每个角色一个 `<div>`，鼠标进入这个矩形就算 hover。但矩形是"假的"——两个相邻角色的透明区域可能互相重叠、遮挡，鼠标其实是在空白处也会误触发。所以要做**逐像素判断**：这个像素上，图片是不是透明的（alpha 值多大）。

## 分四步理解整套流程

**第一步：把每张图"翻译"成一张透明度表**（`buildAlphaMap`）

浏览器给你的 `<img>` 本身不能直接查询某个像素透不透明，得先把图片画到一个看不见的 canvas 上，然后用 `getImageData` 把每个像素的 RGBA 读出来，只留下 A（alpha，透明度）这个通道，存成一个数组。

比如一张 1000×1500 的图，就会得到一个 150 万长度的数组，`alpha[像素索引]` 告诉你这个像素是全透明（0）还是全不透明（255）还是半透明。

这一步只在图片加载完成后做一次，之后不再用 canvas，纯粹查数组，很快。

**第二步：鼠标移动时，判断鼠标在哪个角色的"实心"区域上**（`hitTestAlpha`）

这一步的难点是**坐标换算**，因为鼠标给你的坐标（`clientX/clientY`）是"屏幕坐标"，而 alpha 数组里的坐标是"图片原始像素坐标"（比如 0~999，0~1499），中间要经过好几层换算：

```
鼠标屏幕坐标
  → 减去容器（slot）左上角位置 = 容器内坐标
  → 因为图片是 object-fit: contain（等比缩放居中/贴底显示），
    要先算出图片在容器里实际画在哪个矩形范围（不是整个容器！）
  → 再判断鼠标是否落在这个矩形内
  → 落在矩形内的话，把坐标从"矩形内的比例位置"换算成"图片原始像素坐标"
  → 拿这个像素坐标去查 alpha 数组，看透明度是否够大
```

**第三步：多个角色叠在一起时，按图层顺序（zIndex）从上往下查**

如果有多个角色互相遮挡，要先检查最上层（zIndex 最大）的角色是否命中，命中就直接返回，不用再查下面的——这就是现实中"点鼠标点到最上面那张图"的逻辑。

**第四步：用 requestAnimationFrame 节流**

`mousemove` 事件触发频率可能远超屏幕刷新率（尤其触摸屏/高精度鼠标），如果每次事件都重新做一遍上面的计算，会很浪费。所以用 rAF 包一层：一帧里无论触发多少次 mousemove，只在下一帧真正渲染前算一次。这也是我们上一轮聊的 rAF 的实际用途。

## 四个坑分别在讲什么

| 坑 | 通俗解释 |
|---|---|
| **object-fit: contain** | 图片不是铺满整个容器的，是等比缩放后居中或贴底放的，容器里有"留白"。如果直接拿容器的宽高去做坐标换算，会算错——必须先算出图片"实际画在哪个矩形"，再拿这个矩形做换算基准。 |
| **transform: scale(1.05)** | CSS 视觉上把图片放大了 5%，但是 `getBoundingClientRect()` 这个 API 拿到的是原始未缩放的位置和大小（浏览器不会自动帮你算 transform 后的结果）。所以要手动把矩形按 1.05 倍从中心往外扩一下，才能对上鼠标实际看到的位置。 |
| **视差效果（parallax）也在挪动位置** | 如果你的角色图有跟随鼠标微微漂移的视差动效，那这个角色的位置**每一帧都在变**。所以不能只在组件加载时读一次位置存起来，必须每次鼠标移动时都重新调用 `getBoundingClientRect()` 实时读最新位置。 |
| **图片异步加载** | Next.js 的 `<Image>` 组件图片是异步下载的，如果在图片还没下载完就去 canvas 画图、读 alpha，会读到一张空白图（全透明）。要等 `onLoad` 事件触发，并且确认 `img.complete === true && img.naturalWidth > 0`，才能保证图片数据真的可用。 |

## 代码骨架里三个函数各自的角色

- `buildAlphaMap(img)`：图片加载完调一次，产出一张"透明度地图"，存起来复用。
- `computeContainRect(...)`：纯数学函数，输入容器尺寸和图片原始尺寸，算出图片实际画在容器里的矩形（左上角坐标 + 宽高）。
- `hitTestAlpha(...)`：整合上面两者 + scale 修正，把鼠标坐标一路换算到图片像素坐标，最后查 alpha 数组返回 true/false（命中/未命中）。
- `onMove`：挂在容器上的事件处理器，rAF 节流后，按 zIndex 顺序调用 `hitTestAlpha` 找出当前 hover 的角色，`setState` 触发 `NameCard` 显示对应名字。

## 最后一条建议很关键

它说的"用一个 `<div>` 把 `computeContainRect` 算出来的矩形画成红色边框叠上去看"，是最实用的调试手段——因为坐标换算错了不会报错，只会"感觉不对"，肉眼对比红框和图片实际轮廓，比 console.log 一堆数字直观得多。
