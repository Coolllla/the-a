## 讨论与决策

### 字体导入采用"独立 fonts 模块 + CSS 变量"

在 `app/layout.tsx` 里堆多份 `next/font/google` 声明会膨胀。改为：

- 建 `app/fonts.ts` 集中所有字体导入与 `variable` 声明；
- `layout.tsx` 只 import 拼接后挂 `<html className>`；
- SCSS 里通过 `var(--font-xxx)` 消费。

这样后续加字体只需改 `fonts.ts`，`layout.tsx` 不动。

### `.active` 用 Caveat 手写体

首个真实的字体差异化用途：nav 的 `.active` 项用 `Caveat`（手写连笔体）替换正文字体。配合项目"手绘日记本"色板 + `--highlight-yellow` 的下划线滤镜，形成"当前所在页 = 被手写标记出来"的意象。

### Nav 接入位置：目前塞在 HomeV1（有争议，见"遗留"）

为了立刻能在真实页面上看到 Nav 效果、开始视觉迭代，把 `<Nav />` 直接放进了 `HomeV1.tsx` 的 `<main>` 顶部。**这是体验层，不是外壳层。**

昨天的 log 与 [`decisions/architecture.md`](../decisions/architecture.md) 都明确 nav 属于外壳层（应挂在 `app/layout.tsx` 或路由分组的 `layout.tsx`）。当前这个位置是**预览用的临时接线**，不是最终架构。理由待用户在下一次会话中确认或改回。

## 遗留

### `globals.scss` 的 `font-family` 缺逗号

当前值：`var(--font-geist-mono) system-ui, -apple-system, sans-serif`

`--font-geist-mono` 展开后是 next/font 生成的 font stack 字符串（形如 `"__Geist_Mono_xxx", "__Geist_Mono_Fallback_yyy"`）。跟后面的 `system-ui` 直接空格相连，展开后的第二个字体名会跟 `system-ui` 拼成 `"..._Fallback_yyy" system-ui`，破坏 CSS `font-family` 解析。

正确写法：`var(--font-geist-mono), system-ui, -apple-system, sans-serif`（加逗号）。

另外：把等宽字体 `--font-geist-mono` 作为 body 主字体是否是有意的？考虑正常正文更适合 `--font-geist-sans`。用户下次会话确认。
