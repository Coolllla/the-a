# logs/

跨会话 / 跨设备开发日志。每次和 AI 协作一段有实质产出的工作后，落一份日志到这里，方便下次（无论是自己还是新会话的 AI）冷启动时快速接上。

## 命名约定

`YYYY-MM-DD-<slug>.md`，slug 用短横线小写英文简述本次主题，例如：

- `2026-07-22-nav-skeleton.md`
- `2026-08-01-library-page-mvp.md`

同一天有多次可加序号：`2026-07-22-nav-skeleton.md`、`2026-07-22-2-color-tokens.md`。

## 每份日志建议包含

- **背景 / 会话上下文**：一句话说这次是在什么情境下做的（如"跨设备无美术资产"）
- **讨论与决策**：定下的重要选择，附一句 why
- **写了什么代码**：文件清单 + 每个文件角色一句话
- **留给下一次的接力**：还没做的 TODO、明显的下一步、卡住的地方

写这份日志不是给自己交作业，是给"三个月后忘了上下文的你"或"新一次 AI 会话的 Claude"看的，所以：**决策的 why 比代码的 what 更重要**。代码 diff `git log` 能查，思路只能靠这里留存。

## 与其他 doc 子目录的关系

- `decisions/` = 已稳定的架构/技术决策，正文不改，只追加变更日志
- `notes/` = 单点踩坑与思考，非线性、非索引
- `logs/` = **过程性**记录，按时间流动，读的时候是"最近做了什么、下一步做什么"

如果日志里的某个决策被反复引用、逐渐稳定，就迁移一份到 `decisions/`，日志里保留原文并加"→ 已迁移到 decisions/xxx.md"备注。
